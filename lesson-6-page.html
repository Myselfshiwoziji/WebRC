What is a display declaration?

In CSS, a display declaration is used to define how an HTML element is rendered or displayed on a webpage. It determines the type of box model used for an element, which affects its positioning, layout, and interaction with other elements. The display property in CSS offers several values that can be assigned to an element. Some common values include:

None:

This is a the most simple attribute for the display declaration, as, just like the name suggests, it causes the element that has this declaration to not appear at all. It should also be noted that the usage of the none attribute will also cause the element to not take up any physical space either. When you use this declaration, use the syntax:
display: none;

Block:

When using this attribute in conjunction with the display declaration, it causes the element that was defined to have a block display to create a rectangular block that takes up the entire screen. This rectangular shape might not originally be seen, but when you use the Command Shift C on the element you will see that it has an invisible size of the rectangle. When you use this declaration, use the syntax:
display: block;

Inline:

When you use the inline attribute to the display declaration, it causes the the element that has this declaration to have a size that changes depending on the amount of space that the elements inside of it take up. For example, when you have a div that has this attribute, then the size of the div will only be as large as the total area that the elements inside of it take up. What's interesting about this element is that unlike the block attribute, if there are any extra space left over by the Inline element, then that space can be taken up by other Inline elements. Side note: you can actually add a hyphen to end of the inline and chain on another attribute to have the element take on multiple attributes. When you use this declaration, use the syntax:
display: inline;

Grid:

When you use this attribute, it turns the element that has this declaration into a gridbox. This basically measn that elements inside the container element will be placed in grid like formats based off of how you want it to look like. Once you do turn the element into a gridbox, there will be a variety of different extra declarations and new concepts open for you to use to customise the gridbox's display options, such as:

grid-template-columns:

When you use this declaration in a element that has the display type of a grid, you will be able to customise the width and quantity of grid rows there are in a grid. For example, using grid-template-columns: 200px 200px 200px; would create 3 columns that have the width of 200 pixels. Please note that in a grid format the content inside the grid will be displayed horizontally, and will wrap to a new line when there isnt enought space for a another element to fit inside. Also as another note, you can type grid-template-columns: repeat (number of rows, size of the row); to create multiple identical rows of the same size without needing to constantly type it out.

The fr unit:

Interstingly, when using the grid display declaration, a new unit of measurement is open to be used by you. This unit is quite important when designing grid formats as it allows for precise customisation of the grid rows. To understand this, let's first understand what the fr unit represents. The fr unit is a unit of measurement that basically symbolises the fraction of space in an element that it takes up. For example, if we were to type
grid-template-columns: 1fr 1fr 1fr, we would create 3 rows in our grid element that all each take up 1/3rd of the width of the element.

grid-auto-rows:

However, this declaration controls the opposite of the grid-template-columns declaration, as this declaration controls the height of a gridbox. Just as another side note but all rules that follow the grid-template-columns also apply here (eg using repeat the element wrapping);

For example, if I were to set the row-gap to 5px and the column-gap to 10px, then every gridbox in the grid would have a gap of 5px between each other in the x-axis and 10px in the y-axis.

Flex:

This is one of the most important display attributes you will use, and probably will be the one you will use the most often. The flex attribute to the display attribute turns a element into a highly flexible and customisible container that can easily allow for easy access to a variety of alignment techniques. Much like the grid attribute, using this option for your display attribute allows for new declarations to be used. For example:

flex-direction:

When using this declaration, the content inside the flexbox will change its orientation. Typically, when we make a website, the default settings for arranging elements by a computer is vertically, however, with flex-direction we are able to change it so that the content inside the flexbox can be arranged horizontally. To do this, we can type in flex-direction: row to set the alignment settings in the flexbox to display elements in a horizontal line. As a side note, you can also manually set the flexbox to display elements in a vertical line using the declaration flex-direction: column

❮ Previous Next ❯

align-items:

The align-items declaration is quite useful in a variety of different scenarios, as it allows you to align all of the content in the element to a specific area in the y-axis. The align-items declaration has four main values. These values are, start, center, end, and stretch. The start value for the align-items declaration causes all the items inside the flexbox to be aligned to the top of the flexbox, and the end value fot eh align-items declaration causes all the elements inside the flexbox to be aligned to the bottom of the flexbox. The center value for the align-items declaration, much like what the name suggests, causes the elements inside the flexbox to be aligned to the dead center of the container. Finally, the stretch value causes all the elements to take up all the space in the flexbox evenly.

❮ Previous Next ❯

justify-content:

Much like the previous declaration, the justify-content declaration also controls the alignment of the elements relative to the flexbox itself. However, this declaration controls the alignment of the elements relative to the x-axis of the flexbox. The justify-content declaration has multiple main values that are used to change the alignment of the elements, such as, center, flex-start, flex-end, space-between, and space-around. The flex-start value for the justify-content declaration causes the elements to be aligned towards the left of the flexbox, whereas the flex-end value causes it to be aligned to the right. The center value causes the elements to be aligned in the center of the x-axis. The space-between value causes the elements to have an equal amount of space placed between them, and basically allows for the elements to be evenly spaced such so that it takes up the entire space of the flexbox (Note: this makes it so that the starting element will be placed in the left corner of the flexbox, and the final one will be placed in the right corner.). Finally, the space-around value causes the elements to have some space before and after the element. This basically means that the element has some space in the left and the right of it.

The align-self and the justify-self syntax is another way of re-aligning elements in the flexbox, however, rather than manipulating every single element, the -self syntax allows for the manipulation of a single element in the flexbox and it's own alignment. This is quite useful when creating unique web designs and restructuring your website's content. To use these declarations, you have to put it in the style of a specific element inside a flexbox, and then type in a specific value for this alignment (note: the values in question here are the exact same as the previous two slides).

This is quite a simple declaration, as it simply decides which element goes first in the alignment of a sequence of multiple elements. For example, if I have a red, blue and yellow button, and I give the yellow button the value of order: 1;, then the yellow button will be the first one displayed in the sequence of elements.

For example, if I were to set the row-gap to 5px and the column-gap to 10px, then every gridbox in the grid would have a gap of 5px between each other in the x-axis and 10px in the y-axis.

What is the position declaration:

In CSS (Cascading Style Sheets), the "position" declaration is used to specify how an element is positioned within its containing element or the web page as a whole. It controls the layout and placement of elements on the page. The "position" property can have several values:

Static:

This is the most basic position value, and is actually the default setting for the position of you don't change the positon value yourself. The static value for position basically means that the element itself can easily be manipulated on where its location on the webpage, and simply follows the normal flow of the webpage. When you use this declaration, use the syntax:
position: static;

Fixed:

This value for the position declaration causes the element to be stuck to the screen regardless of where the user scrolls to on the webpage. An example of this would be the youtub search bar, and how it always sticks to the top of the Youtube homepage. When manipulating the placement of the fixed element, you can use the declarations:

• top

• bottom

• left

• right

These declarations allow you to manipulate how far the element is from the its container element's top, bottom, left and right. For example, if I were to type left:0; right: 0; it would cause the element to be 0px away from the right hand side corner.

Absolute:

This value for the position declaration is shares alot of similarities to the Fixed vale, however, the unlike the fixed value, the absolute value only sticks itself to a fixed position inside of an element rather than on the screen itself. An example of this would be the video length display box on every Youtube video when you boot up the homepage. Much like the fixed value, the top, bottom, left, and right declarations are also used to manipulate where the element will be fixed inside of its parent element. When you use this declaration, use the syntax:
position: absolute;

Sticky:

This value for the position declaration is very different from the other values, as it is sort of like a combination of them. This value for the declartion allows for you to make elements stick to your screen when you scroll pass them. For example, if I were to scroll down a page, and the top of my webpage touches an element with the sticky position, then it will be stuck there similar to that of an fixed value position element. However, when I scroll back to the place where the sticky element was taken from, it returns back to its original position, similar to that of a static element. When you use this declaration, use the syntax:
position: sticky;
