Measurements using CSS:

There are a variety of different units of measurements for CSS, and knowing which one to use for different occasions can greatly change productivity and effieciency when designing a webpage. For the purposes of this course, we will only talk about the ones which we consider to be the most useful, for example:

Pixels:

This is one of the most common unit of measurement of CSS, and the unit of measurement we have been using in CSS for a long time now. To understand the measurement system of pixels, you should first understand that every single digital file, screen, etc, are all made of thousands upon thousands of pixels with sligntly different colors to make the illusion of what you see. Therefore, the measurement of pixels refers to the number of pixels on the screen that it takes up. The shortened version of this unit of measurement is px.

Centimeter:

Much like real life centimeters are also another valid unit of measurement in web development. The centimeter used in web development and in real life are the same, so you don't need to worry about it being different. This unit of measurement is denoted with cm.

em:

This unit of measurement is much more difficult to explain, as this unit of measurement does not have a direct basis in which it is based upon. To put it simply, this unit of measurement is relative to the size of the current font size of the element. For example, 2em would mean that the text would be 2 times larger than the base font size of the element.

vw:

This unit of measurement is constantly changing in its value, however, this unit of measurement basically shows the width of the element relative to the width of the viewport in terms of percentage. For example, 3vw would mean that the element has a width that is the size of 3% of the viewport. If you are confused as to what a viewport is, a viewport is basically the current size of the browser that you are looking at based off of the electronics you are using.

vh:

This unit of measurement follows the exact same logic of the previous unit of measurement, however, this unit of measurement instead is referring to the height of both the viewport and the element.

vmin:

This unit of measurement is similar to vh and vw, however, it instead refers to the size of the element relative to the smallest possible viewport size (basically if you shrink a chrome tab to its smallest point). For example, 1vmin would refer to 1% of the smallest possible viewport size.

vmax:

This unit of measurement refers to the size of the element relative to the largest possible viewport size (basically if you enlarge a chrome tab to its largest point). For example, 1vmax would refer to 1% of the largest possible viewport size.

What are pseudo-classes?

In the previous lessons, we had discussed the usages of classes and identifying them to customise their properties in CSS. However, there is one major type of class that we had not discussed, which is the pseudo-class. Pseudo-classes add an extra layer to the customization of elements in CSS, as they add specific conditions for when the element should change properties. For example, a very common pseudo-class would be the :hover pseudo-class, which makes it so that when an element is being hovered over it should change its properties. To use a pseudo-class, simply type your regular CSS setup, however, next to the name of the element or class of the element type in : and a pseudo-class to give specific properties when the conditions of the pseudo-class are achieved. As for some examples of pseudo-classes, here are some below:

:Hover

This is by far the most common pseudo-class, and will be one which you will get quite familiar with. Much like the name suggests, properties (or delcarations) put into any CSS code that is under the pseudo-class of :Hover will only be applied to the element if it is being hovered over with by the user's cursor. To use this pseudo-class, you can follow the format below:
.exampleClass:hover{Type in declarations here}

:Active

This is another incredibly common pseudo-class, and it basically states that the delcarations inside this pseudo-class will only be applied when a element is clicked or has any sort of interation with it. To use this pseudo-class, you can follow the format below:
.exampleClass:active{Type in declarations here}

:nth-child(n)

At first, this pseudo-class might seem weird and complicated, but it is quite easy to understand. To put it simply, let's say we have 15 or so of the same element in the same page, but we want the 6th element to be different, then we would use :nth-child:(6) to directly identify the 6th version of this element in the list and customise it. Please note that the n used in the title of this part of the website is simily filler, and you have to either replace it with a number, the word even (to denote that the trigger is every even child of the element), or the word odd (to denote that the trigger is every odd child of the element). Regardless, to use this pseudo-class, you can follow the format below:
.exampleClass:nth-child(n){Type in declarations here}

:focus

At first glance, this pseudo-class might seem like the same thing as active, when in reality it is quite different. Typically, the :focus pseudo-class is used for input type elements, as its trigger condition when the element is is focused upon. This is different to active as when you click on something with the :focus pseudo-element, its trigger condition is only unfurfilled when the user clicks on another element or somewhere else in the page. To use this pseudo-class, you can follow the format below:
.exampleClass:focus{Type in declarations here}

:visited

This is a very simple pseudo-class, and it basically states that if the link in the element was visited before, then the trigger conditions for this pseudo-class will be furfilled. To use this pseudo-class, you can follow the format below:
.exampleClass:visited{Type in declarations here}

:link

This is basically the opposite of the previous pseudo-class, as it the trigger conditions for this pseudo-class is simply just links that have not been visited yet. To use this pseudo-class, you can follow the format below:
.exampleClass:link{Type in declarations here}

More CSS Declarations

Opacity:

Opacity is a CSS declaration that, just like the name suggests, toggles how transparent the element is. This transparency is on a scale of 0 to 1, where 0 is completley transparent and 1 is completley solid. When you use this delcaration, use the syntax:
opacity: insert opacity level here;

Transition:

The transition declarations takes care of how elements transition from one declaration state to another. This delcaration works by setting the time taken (in seconds) to transition from one declaration to another. When you use this declaration, use the syntax:
transition: 0.2s;

Text alignment:

The text alignment declaration, just like the name suggests, is basically the declaration in charge of changing how the text is aligned. For example, text alignment center would change the text to be aligned to the center of the element. When you use this delcaration, use the syntax:
text-align: left; OR right; OR center;

(Please note that you can only have on attribute for text-align, and can't do something like left center.)

Text decoration:

The text decoration declaration is mainly in charge of adding extra attributes to the text in an element. For example, you can add an overline, a underline, and a line-through a text. When you use this declaration, use the syntax:
text-decoration: overline; OR underline; OR line-through;

Text spacing:

When we talk about text spacing in CSS, there are multiple different declarations that can be used for text spacign that all achieve different things, however, for the sake of this class, we will only focus on line height. Line height is a declaration that basically states how much space 1 line of text takes in an element. This space is mainly defined through the usage of the units of measurement discussed before. When you use this declaration, use the syntax:
line-height: MeasurementValue;

Text shadow:

Text shadow is, compared to the other declarations, quite unique in its own way. Text shadow adds an extra copy of the text in an element, and using the text shadow declaration you can change the displacement of the copy relative to the element, add blurriness to the element and the text color of the copy. When you use this declaration, use the syntax:
text-shadow: xDisplacement yDisplacement blurriness color

(Please note that all values, excluding color, are all denoted in the CSS unit of measurements stated before.)
