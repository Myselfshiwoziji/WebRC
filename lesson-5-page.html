Animations

Animations in HTML and CSS are techniques used to bring elements on a web page to life, creating movement and visual effects. They allow developers to add dynamic and interactive elements to their websites, enhancing the user experience. In this course, we will teach you to use @keyframe, and from ... to ... to display simple animations.

Making animations

Method 1: Keyframe percentages

Keyframes are static frames where something happens. These will serve as your pivot points to manipulate your animation around, with the animations essentially being the smooth connections between the keyframes.

To make a keyframe, you insert a @keyframe in CSS, then selecting your name, and assign what you want to keyframe to do. Use percentages to indicate what you want the keyframe to do at a specific completion precentage (starting at 0% and ending at 100%) For example:
@keyframes example {
 0% {background-color:rgb(0,0,0);}
 34% {background-color:rgb(255,0,0);
   font-size:30px;}
 67% {background-color:rgb(0,255,0);}
 100% {background-color:rgb(0,0,255);}
}
#animation {
 animation: example 2s linear infinite
}

To reference the animation, just insert the id into the anchor/ paragraph/ otherwise that you want the animation to be played on.
for example, using the keyframes above:
<a id = 'animation'>hi!</a>

id's with animations

Has you have seen above, you can see that we have used an ID to link the animation to the text, which is one of the main features of IDs, that they can bring properties of the ID to all of the elements that have the ID.

Method 2: From ... To ...

From the text itself, it should be self explanatory on how this method works. Once you play the animation, it will go from 'from' all the way to 'end'. See the example below:
@keyframes slidein {
 from {
  margin-left: 100%;
  width: 300%;
 }
 to {
  margin-left: 0%;
  width: 100%;
 }
}
Then, simply insert your animation to your desired element in the way that you did with keyframes. Note that you would assign properties of the animation like speed, repeats, length, etc. in the ID and not the animation itself. Both methods work well, so just decide on which that you want to use. FOr the purpsoes of this lesson, most of the practices will be using method 1, but note that it is entirely valid to use method 2 as well.

Assigning HTML to animations % examples

Contrary to the name, if you want to assign html attributes, it would be easier to do it in CSS. To do it in CSS, you can:

Duration

assigning the animation-duration attribute to an animation can control its duration. (make sure to specify whether its in miliseconds or seconds)

Timing

This defines how fast an animation is played during the time which it is played. The prompts can include ease, which is a prompt where it goes from slow to fast exponentially (ease-in, ease-out, and ease-in-out), and linear where it goes from slow to fast in a linear manner.


Heres an example of how different timings change, given the same animation as above (take note that all of these animations are 2s long):

Iterations

Iterations is the number of times an animation is played. using animation-iteration-count, you can control the amount of times its played, from 1 to infinite for continuous looping. Or, you can just do it in the id with the syntax animation: [name] [duration of animation] [Timing] [iterations] For example, with the example above, you can see that the name is example, the animation is 2s long, it goes linearly to completion, and it loops infinitely.

Other customisation commands

In fact, all other HTML attributes works in animations as well! The only difference is that the object will return to the original state after the animation is played. Such, other commands such as displays, margins, scale, position, etc. works just as the same as if you were assigning a class.
