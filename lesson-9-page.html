Events in JavaScript

What is an event in JS?

In JavaScript (JS), events are actions or occurrences that happen in the user interface or the browser. These events can be triggered by various actions, such as a user clicking a button, pressing a key, or the browser finishing loading a web page.

Event Listeners

An event listener is required if you want your program to recognise that you called an event to execute a function. For every function that you put, you will have to add an event listener to an event in JS (for example, the clicking of a button). In short, Event listeners "listen" for specific events and execute a function when the event occurs.

Types of Events in JS

   onclick()

The onclick listener in JavaScript is an event handler that is used to execute a particular piece of code when an element, such as a button or a link, is clicked by the user. It is commonly used to create interactivity and trigger actions in response to user input.

   mousedown()

Occurs when the mouse button is pressed down. This is different from onclick() because it detects if a mouse button is being held down instead of clicked.

   mouseover()

Occurs when the mouse pointer enters an element.

   keydown()

Occurs when a key is pressed and held down.

   keypress()

Occurs when a key is pressed down and released.

   load()

Occurs when the page/document has finished loading. This is primarily for loading screens, which we will cover in one of our extra lessons (Lesson 11 about animations)

Parameters

In JavaScript, parameters are the values passed to a function when it is called. A function can have zero or more parameters, which are defined in the function's declaration. Note that parameters in JavaScript are optional, meaning that you can define a function with no parameters with empty brackets (for example, onclick())

  What are parameters used for?

In JavaScript, parameters are used in function definitions to specify the variables that a function expects to receive when it is called. Parameters act as placeholders for values that will be passed into the function.
Function parameters allow you to pass values to a function so that you can use and manipulate those values within the function's code. They define the inputs that a function expects, and you can specify multiple parameters separated by commas.

Uses and examples of parameters

Creating reusable and adaptable code:

By using parameters, you can create functions that can work with different data or conditions. This makes your code more modular, reusable, and adaptable to different scenarios.

Here is an example: Suppose you are making a quiz and want to make a hint button (with an id called Hintbutton) to display a hint (with id Hintparagraph), so you type:
 
function buttonclick() {
    for (let i = 0; i < document.querySelectorAll(#Hintbutton).length; i++) {

      } else {
        paragraph.style.display = 'none';
        paragraph.style.visibility = "hidden";
      }
    }

However, you find that if you click any button in the iframe, all paragraphs with the id paragraph will show.

so instead, using parameters, you can configurate the code accordingly:

const hint_texts = document.getElementsByClassName("hint-text");
function buttonclick(hint_id){
    for (let i = 0; i < hint_texts.length; i++) {
        let hint = hint_texts[i];
        let display_status = hint.style.display;
          if (i == hint_id) {
            if (display_status == 'block'){
              hint.style.display = 'none';
              hint.style.visibility = "hidden";
          }
          else{
            hint.style.display = 'block';
            hint.style.visibility = "visible";
        }
      }
    else{
      hint.style.display = 'none';
      hint.style.visibility = "hidden";
    }
  }
}

In your HTML, you can then configure each button to run the function with a different hint_id:

<button class="hint-button" onclick="buttonclick(0)">Hint</button>

Declaring a variable in JS

there are 3 main types of variables you will use in JS: Var, Let and Const.

Var:

Variables declared with var are function-scoped, meaning that they are accessible within the function where they are declared, as well as any nested functions within that function. However, variables declared with var are not block-scoped, which means they can also be accessed outside of the block where they are declared. It's worth noting that the var keyword has some characteristics that can lead to unexpected behavior, such as variable hoisting and the lack of block scope.

Let:

Variables declared with let are similar to variables declared with var in terms of their ability to store values. However, the key difference is that variables declared with let are block-scoped, meaning they are only accessible within the block where they are declared. Unlike var, variables declared with let are not hoisted to the top of their scope. This means that you cannot access a let variable before it has been declared in your code.
 
Thus, using let allows for better control over variable scope and can help prevent unintended issues that can occur with var. Therefore, it is generally recommended to use this instead of var when declaring variables, especially in modern JavaScript development.

Const:

When a variable is declared with const, its value cannot be changed once it is assigned. This makes const variables suitable for storing values that should remain constant throughout the execution of a program. Like variables declared with let, const variables are also block-scoped. They are only accessible within the block where they are declared and they are not hoisted to the top of their scope. However, it's important to note that while the value of a const variable cannot be reassigned, it does not mean that the value itself is immutable. If a const variable holds an object or an array, the properties or elements of that object or array can still be modified, but the variable itself cannot be reassigned to a different object or array.

Overall, using const helps convey the intent that a variable should remain constant, which can enhance code readability and prevent accidental reassignments. Thus, it is recommended to use const instead of var or let when declaring a variable that do not need to be reassigned.

Which to Use?

It is best practise to first attempt to use const to prevent accidentally reasigning variables that won't be changed. If the variable will be changed, try to use let to keep the scope of your variable to your block. This will prevent accidentally accessing an old variable outside the relevant area if you have variables with similar names. Try to only declare variables with var if necessary.
